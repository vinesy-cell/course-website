# 网站机器人后台操作说明

## 入口

- 后台：`http://localhost:4174`
- 网站预览：`http://localhost:4174/preview/`

后台会自动读取 `/Users/maxlee/Documents/LK ThinkNote/07_课程网站`。

## 四个按钮

### 同步内容

从 Obsidian 中读取当前状态、首页内容骨架和公开联系方式，生成 `content/site-data.json`，同时复制网站要用的二维码和海报。

### 构建网站

同步最新内容，生成 `dist` 静态网站，并检查主标题、联系方式、图片和禁用口径。

### 打开预览

打开当前本地网站。发布前先检查电脑端与手机端。

### 发布上线

同步、构建、检查、Git 提交并推送到远程仓库。远程仓库接入 GitHub Pages 后会自动部署。

## 推荐日常流程

1. 在 Obsidian 中更新课程网站资料。
2. 等待机器人自动同步和构建。
3. 打开后台，确认最近同步时间。
4. 打开预览，检查文字、图片和二维码。
5. 点击“发布上线”。

## 自动模式

运行 `npm run robot:install` 后，机器人会作为 macOS 后台服务持续运行，电脑重启后自动启动。

- 当前电脑已于 2026-06-08 安装并启动该常驻服务。
- 默认：自动同步和构建，手动点击发布。
- 全自动上线：远程仓库配置完成后，将 `site.config.json` 的 `autoPublish` 改为 `true`。
- 服务日志：`/Users/maxlee/Documents/网站/.robot-service.log`
- 停止并移除常驻服务：`npm run robot:uninstall`

## 发生问题时

先看后台“机器人日志”。也可以运行：

```bash
npm run content:sync
npm run build
npm run check
```

恢复入口是仓库根目录的 `task-progress.md`。
