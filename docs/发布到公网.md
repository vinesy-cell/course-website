# 发布到公网（GitHub Pages）

> 本文件是一次性配置说明。配置完成后，日常发布只需点击后台"发布上线"按钮。

---

## 前提条件

- 有 GitHub 账号（没有请先在 https://github.com 注册）
- 本机已生成 SSH 密钥（已完成，见下文）

---

## 第一步：复制公钥，添加到 GitHub

**公钥内容**（已生成在本机 `~/.ssh/id_ed25519_course.pub`）：

运行以下命令查看并复制：

```bash
cat ~/.ssh/id_ed25519_course.pub
```

然后：

1. 打开 GitHub → 右上角头像 → **Settings**
2. 左侧选 **SSH and GPG keys**
3. 点击 **New SSH key**
4. Title 填：`maxlee course site`
5. Key 粘贴上面复制的内容
6. 点击 **Add SSH key**

---

## 第二步：在 GitHub 创建空仓库

1. 打开 https://github.com/new
2. Repository name 填：`course-website`（或任意名称）
3. 选 **Public**（GitHub Pages 免费计划需要 Public）
4. **不要**勾选任何初始化选项（README、.gitignore 等均不勾）
5. 点击 **Create repository**

记住你的 GitHub 用户名和仓库名，例如：
- 用户名：`maxlee`
- 仓库名：`course-website`

---

## 第三步：运行配置脚本

打开终端，进入网站目录后运行：

```bash
cd '/Users/maxlee/Documents/LK ThinkNote/08_课程网站/06_网站工程'
node scripts/setup-github.mjs <你的GitHub用户名> <仓库名>
```

例如：

```bash
node scripts/setup-github.mjs maxlee course-website
```

脚本会自动完成：
- 测试 SSH 连通性
- 添加远程仓库
- 推送全部代码

---

## 第四步：开启 GitHub Pages

1. 打开你的仓库主页，例如：`https://github.com/maxlee/course-website`
2. 点击 **Settings**（顶部标签）
3. 左侧选 **Pages**
4. Build and deployment → Source 选择：**GitHub Actions**
5. 保存

GitHub Actions 会自动开始构建，约 1-2 分钟后网站上线，地址为：

```
https://<用户名>.github.io/<仓库名>/
```

---

## 配置完成后的日常流程

1. 在 Obsidian 更新课程内容
2. 打开后台 http://localhost:4174
3. 点击"发布上线"

或在终端运行：

```bash
cd '/Users/maxlee/Documents/LK ThinkNote/08_课程网站/06_网站工程'
npm run publish
```

---

## 开启全自动发布（可选）

如果不想每次手动点击，可编辑 `site.config.json`：

```json
"autoPublish": true
```

保存后，机器人检测到内容变化时将自动发布。

---

## 故障排查

| 问题 | 解决方法 |
|------|----------|
| `Permission denied (publickey)` | 重新检查步骤一，确认公钥已添加到 GitHub |
| `Repository not found` | 确认仓库名拼写，或仓库是否选了 Public |
| GitHub Actions 构建失败 | 打开仓库 → Actions 标签查看日志 |
| 网站显示 404 | 确认 Pages Source 已选 GitHub Actions，等待约 2 分钟 |
